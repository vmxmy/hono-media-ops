# 快速创作素材滑选设计

## 目标与范围
- 目标：将“快速创作”表单中的“参考文章 URL”替换为“当前素材选择”，支持搜索并以唱片式左右滑动卡片选择素材。
- 范围：仅改动快速创作入口表单与对应交互组件；复用现有素材数据接口与状态管理。
- 成功标准：用户能在表单顶部通过搜索+滑动选中素材，提交时使用 `refMaterialId`，无 URL 输入。

## 交互与界面结构
- 顶部区域替换为“当前素材选择”区块。
- 区块结构：搜索输入框 + 唱片式滑动卡片 + 选中素材摘要。
- 交互：触摸/鼠标拖拽/键盘左右切换；选中状态清晰可见。
- 视觉：圆形或近圆形卡片、中心孔与柔和阴影；切换时轻微旋转与倾斜，强调唱片感。
- 空态：无素材或搜索无结果时提示并引导去素材仓库添加。

## 数据流与状态
- 数据来源：复用 `reverseLogs.getAll`，支持 `search` 参数过滤标题/风格名等字段。
- 状态：`materialSearchQuery`、`materials`、`selectedMaterialId`。
- 搜索更新：搜索词变化后请求新列表，并将选中项重置为第一条（若存在）。
- 表单提交：仅携带 `refMaterialId`，移除 `sourceUrl` 字段。

## 错误处理与性能
- 查询失败：显示轻量错误提示并提供重试。
- 性能：默认一页（例如 50 条），滑动组件仅渲染当前项与相邻项，图片懒加载。
- 体验：保留上一次成功结果以应对临时错误或慢网。

## 测试要点
- 搜索后列表更新且选中项正确重置。
- 滑动/键盘切换能更新 `selectedMaterialId`。
- 空态、错误态 UI 正确出现。
- 提交时 `refMaterialId` 正确传递。

## 工程原则对照
- KISS：复用现有查询与滑选组件，避免复杂 3D 与多入口。
- YAGNI：不做分页滚动与高级缓存，按当前需求实现。
- SOLID：搜索、选择、展示职责分离。
- DRY：复用素材数据结构与卡片渲染逻辑。
